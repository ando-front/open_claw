# OpenClaw 日常の利用ガイドと運用手順

このガイドは、OpenClaw（Mac Mini 内の Google Gemini システム）を日頃どのように活用するか、運用・トラブルシューティングの方法についてまとめたものです。

---

## 1. LINE からの日常的な操作
スマホの LINE から、あらかじめ承認した公式アカウント（open_claw）に対してチャットを送ると、バックグラウンドの OpenClaw がその内容を処理します。

### a) 会話形式での使い方
* **「こんにちは」「自己紹介して」** など、普通に会話を行うことができます。
* **「いま Mac の画面に出ているのは何？」** など、OS 環境に基づいた質問ができる場合もあります。（設定しているツール・スキルの権限によります）

### b) コマンドと設定
* **「明日の予定をカレンダーに追加して」「この URL を要約して」** といったタスクの代行依頼。
* **「~/.openclaw/logs を表示させて」** 等のコマンド実行（※過度な権限が付与されている場合はセキュリティ警告が出ます）。
* Web 上の最新ニュースや情報を調べるなど、ブラウザ操作（`web_search`）や情報検索の代理。

### c) タイムアウトについて
* ボットが考え込んでいる（処理に時間がかかっている）間に、**「まだ？」などと急かすと処理が終了してしまう** ことがあります。
* 必要なタスクの処理中はおとなしく待ちます。処理が終わると必ずレスポンスが届きます。

---

## 2. Web UI (TUI および Dashboard) の活用

LINE からだけでなく、Mac Mini 上のブラウザやターミナル（テキストUI）からも直接 OpenClaw と対話できます。

### TUI (Terminal User Interface) の起動
ターミナルで対話モード（チャットUI）を起動します。
```bash
openclaw tui
```
* **終了方法**: `Ctrl + C`（1回押しただけでは終了しません、2回連続で押します）。

### ブラウザダッシュボードの表示
```bash
openclaw dashboard
```
* ブラウザが自動的に開き、グラフィカルな管理画面（ポート 18789 など）が表示されます。設定や動作ログの確認はこちらが便利です。

---

## 3. バックグラウンドサービスの自動化（再起動時の運用）
Mac Mini が再起動した際、自動的に OpenClaw の「脳」である Gateway サービスが復帰するように設定しておくのが理想的です。

1. **Mac 起動後の手動開始方法**
   ```bash
   sudo tailscale up
   sudo tailscale funnel 18789
   openclaw gateway --force
   ```
2. **自動起動 (LaunchAgent 等)**
   * `launchctl` 等を利用し、ログイン時に自動的に `openclaw gateway` のバックグラウンドプロセスを走らせる設定を行うと快適です。

## 4. トラブルシューティング（FAQ）

### Q: LINE から送っても全く返信が来ない（既読もつかない）
1. **公開 URL（Tailscale Funnel）の接続確認**
   ターミナルで以下を叩き、URL が表示されるか確認してください。
   ```bash
   tailscale funnel status
   ```
   もし `No serve config` などと出ているなら、次を実行してトンネルを張り直します。
   `sudo tailscale funnel 18789`
2. **LINE Bot の Webhook 設定を確認**
   Developer Console 等で、「Webhook の利用」が「オフ」になっていないか、Webhook URL に間違いがないか・「検証」が「成功」するか確かめてください。

### Q: LINE の返信が「申し訳ありませんが個別のお問い合わせは...」になってしまう
LINE Official Account Manager（設定 ＞ 応答設定）で **「応答メッセージ」がオン** になっています。これを必ず **オフ** にしてください。

### Q: 「アクセスがブロックされました（openclaw:access not configured）」というエラーになる
LINE に追加された新しいご家族や友人などを、OpenClaw がセキュリティのためにシャットアウトしています。
ターミナルを開き、以下のコマンドで承認してください。
```bash
openclaw pairing list           # 承認待ちのリストと「コード」を確認
openclaw pairing approve [CODE] # 送信者を許可する
```

### Q: ログを確認したい
システムの不具合や詳細なアクセス履歴は、一時ログ等に出力されます。
一番新しいログファイルを見る場合は以下のコマンドです。
```bash
tail -n 100 /tmp/openclaw/openclaw-$(date +%Y-%m-%d).log
```
※または、Dashboard の画面からリアルタイムの接続ログが確認できます。
